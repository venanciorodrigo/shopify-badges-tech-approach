Technical Approach

Overview

The goal of this implementation is to build a Badge Rules Engine using Shopify Metaobjects, Metafields, and Shopify Flow.

The engine should be platform-agnostic, allowing it to work with both Shopify Headless storefronts and Shopify Native themes. The engine itself is responsible for defining badge rules and configuration, while the storefront implementation (snippets for native Shopify or frontend components/functions for headless implementations) is responsible for evaluating those rules and rendering the appropriate badges.

Data Model

The first step is to create a Badge metaobject with the following fields:

Field Type Required Description
Label single_line_text ✅ Text displayed on the badge
Start Date date_time No Badge is not rendered before this date/time
End Date date_time No Badge is not rendered after this date/time
Collections list.collection_reference No Render only for products belonging to any of these collections
Inventory Min integer No Render only when total product inventory is greater than or equal to this value
Inventory Max integer No Render only when total product inventory is less than or equal to this value
Priority integer ✅ Determines render order. Lower values have higher priority (1 = highest priority)

Product Metafield

After creating the Badge metaobject definition, create a Product metafield that references the Badge metaobject.

Merchants can then associate one or more badges with a product directly from the Product Metafields section. At runtime, the storefront evaluates each badge's rules and renders only those whose conditions are satisfied.

Badge Taxonomy

The following badges are expected to be supported:

Badge Logic
New product.created_at is within the last N days
On Sale compare_at_price > price
Best Seller Product belongs to the Best Sellers collection
Limited Manually assigned
Staff Pick Manually assigned
Seasonal Manually assigned or controlled by Start Date / End Date
Back in Stock Managed automatically through Shopify Flow when inventory changes from 0 to >0

Code samples

Create a new snippet: snippets/product-badges.liquid.

Render it {% render 'product-badges', product: product %} where product badges are needed, for example in product-card.liquid (PLP) and main-product.liquid (PDP): 

<!-- Get all badges ordered by priority -->
assign all_badges = shop.metaobjects.badge.values | sort: 'priority'

<!-- Loop through all badges -->
{%- for badge in all_badges -%}

  <!-- On Sale -->
  if product.compare_at_price > product.price
    ...
  endif

  <!-- New -->
  assign created = product.created_at | date: '%s' | times: 1
  assign age_days = now | minus: created | divided_by: 86400
  if age_days < 30
    ...
  endif

  <!-- Product badge metafield -->
  for ref in product.metafields.custom.badges.value
    if ref.handle == badge.handle
      ...
    endif
  endfor

  <!-- Date filters -->
  if badge.start_date
    ...
  endif
  if badge.ends_at
    ...
  endif

  <!-- Collection filter -->
  assign product_collection_ids = product.collections | map: 'id'
  for c in badge.collections.value
    if product_collection_ids contains c.id
      ...
    endif
  endfor

  <!-- Inventory filters -->
  if badge.inventory_min
    ...
  endif
  if badge.inventory_max
    ...
  endif

{%- endfor -%}

<!-- Render matching badges -->
{%- if rendered_items | strip != '' -%}
  <ul class="product-badges">{{ rendered_items }}</ul>
{%- endif -%}

Note: If the storefront is headless (e.g., Next.js, Hydrogen), we can encapsulate this logic in shared utility files or helper methods and reuse them wherever needed, such as on the PLP and PDP.

Assumptions

1. Shopify Flow

This approach assumes Shopify Flow is available.

For stores without Shopify Flow, there are two possible alternatives:

Build a lightweight custom shopify APP (for example, using AWS Lambda and Shopify Webhooks) that listens for product and inventory updates and automatically manages badge assignments by updating the product metafields.

Or, if we think this would overengineer the solution, we could leave this tag as a manual process for non-Shopify Plus stores.

2. Badge Styling

Styling is not stored in the Badge metaobject, but it can be added in two ways:

Simple approach: add a new field (e.g. Style) with predefined values such as Primary, Secondary, Dark, Gray, Orange, etc.
More flexible approach: create a separate Style metaobject containing presentation attributes such as background color, text color, icon, border, link and other visual properties. The Badge metaobject would then reference a Style, allowing design changes without modifying badge definitions.

Add'I writeup

Back in Stock: How long should the badge persist after restock?
Priority: If more than two badges have the same priority, how should they be ordered? Should we sort them alphabetically or display them in a random order?
Visible badge cap: What should the default maximum number of visible badges be (e.g., 3 or just 1)? If the limit is reached, should system-generated badges (such as On Sale or Back in Stock) take priority over manually assigned badges?
Migration of existing hardcoded badges: Conduct an audit and cleanup of existing hardcoded badges. If needed, develop migration scripts.
